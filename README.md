Короткий опис розробленої програми

Програма реалізована мовою C# на платформі .NET 8 у вигляді консольного застосунку. Вона розв'язує задачу лінійного програмування методом модифікованих жорданових виключень (МЖВ). Алгоритм складається з двох фаз: пошук опорного (допустимого) розв'язку та пошук оптимального розв'язку. На кожній ітерації програма виводить симплекс-таблицю з позначками змінних, рядок і стовпець розв'язувального елемента та перераховані значення.

Структура програми

Програма розбита на 11 класів.

Program - точка входу. Містить головне меню з чотирма пунктами (пошук опорного розв'язку, пошук оптимального розв'язку, максимум/мінімум функції мети Z, тестові обчислення), яке з'являється повторно після кожної операції.

LpProblemInput - зчитування постановки задачі з консолі. Запитує кількість змінних і обмежень, напрямок оптимізації (max/min), коефіцієнти функції мети та кожне обмеження у форматі "a1 a2 ... an  знак  b".

TestRunner - автоматичне виконання тестових прикладів на захардкожених даних без участі користувача: варіант завдання та два приклади з методички.

JordanElimination - процедура МЖВ (рисунок 2.1). Розв'язувальний елемент замінюється на 1/a_rs, елементи рядка діляться на нього, елементи стовпця діляться зі зміною знаку, решта перераховується за формулою b_ij = a_ij - a_is * a_rj / a_rs.

SimplexTable - структура симплекс-таблиці. Зберігає саму матрицю, позначки стовпців (-x1, -x2, ...) і рядків (y1, y2, ...). Містить метод SwapLabels, який міняє місцями позначки розв'язувального рядка та стовпця після МЖВ.

ProblemBuilder - приводить усі обмеження до вигляду "... >= 0" (нерівності <= домножаються на -1) і будує вхідну симплекс-таблицю. Виводить переписану систему обмежень у протокол.

BasicSolutionSolver - пошук опорного розв'язку (рисунок 2.3). Шукає від'ємний елемент у стовпці вільних членів, у його рядку - від'ємний елемент, це розв'язувальний стовпець. Далі за мінімальним невід'ємним відношенням обирає розв'язувальний рядок і викликає МЖВ. Завершується, коли всі вільні члени невід'ємні, або повідомляє про суперечливість системи.

OptimalSolutionSolver - пошук оптимального розв'язку для задачі максимізації (рисунок 2.4). Шукає від'ємний елемент у Z-рядку - це розв'язувальний стовпець. Розв'язувальний рядок обирається за мінімальним невід'ємним відношенням b_i / a_ij серед додатних елементів стовпця. Завершується, коли Z-рядок не містить від'ємних елементів, або повідомляє про необмеженість функції мети.

MinToMaxConverter - перетворення задачі мінімізації на задачу максимізації (рисунок 2.2 б). Просто змінює знаки коефіцієнтів функції мети: Z' = -Z. Після розв'язання максимум Z' дорівнює мінус мінімуму Z.

LinearProgrammingSolver - оркестратор (рисунок 2.2 а). Виводить постановку задачі, при потребі перетворює min на max, будує таблицю, викликає фази пошуку та витягає значення змінних з базису. Залежно від режиму SolveMode зупиняється після опорного розв'язку, виводить оптимальний розв'язок або додатково друкує фінальне Max(Z)/Min(Z).

Protocol - форматований вивід протоколу. Друкує симплекс-таблицю з вирівняними колонками, позначки розв'язувальних рядка та стовпця, опорний та оптимальний розв'язки й підсумкове значення функції мети.

Вхідні та вихідні дані

Для пунктів 1-3 меню користувач вводить кількість змінних n та кількість обмежень m, напрямок оптимізації (max або min), n коефіцієнтів функції мети через пробіл та m обмежень у форматі "a1 a2 ... an  знак  b", де знак - це <=, >= або =.

Пункт 1 виводить протокол пошуку опорного розв'язку та сам опорний розв'язок X = (x1, ..., xn). Пункт 2 додатково виконує фазу пошуку оптимуму і виводить оптимальний розв'язок. Пункт 3 до всього цього додає фінальне значення Max(Z) або Min(Z).

Пункт меню "Тестові обчислення" виконує повне розв'язання на трьох заздалегідь підготовлених задачах: варіант завдання Z = X1 + X3 + X6 -> max та два приклади з методички. Варіант дає X = (2.50; 0; 0; 0; 0; 0.50), Max(Z) = 3.00.

Елементи рядка вводяться через пробіл. Числа приймаються як через крапку, так і через кому. Протокол обчислень виводиться безпосередньо в консоль.
